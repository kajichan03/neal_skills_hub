# GitHub Bug 创建助手

通过交互式对话收集 Bug 信息，创建格式规范的 GitHub Bug Issue。

## 触发条件

用户提到以下关键词时触发：
- "提交 bug" / "创建 bug"
- "记录问题" / "报告错误"
- "开个 bug issue"
- "记录一下这个 bug"

## 信息收集流程

### Step 1: 收集必需信息

按顺序询问用户以下信息：

1. **问题标题**（一句话概括）
   - 示例："登录页面无法提交表单"
   
2. **问题描述**（发生了什么）
   - 示例："用户在点击登录按钮时页面无响应"
   
3. **复现步骤**（如何触发）
   - 示例：
     ```
     1. 打开登录页面
     2. 输入正确的用户名和密码
     3. 点击"登录"按钮
     ```
     
4. **期望结果**（应该发生什么）
   - 示例："页面应跳转到用户主页"
   
5. **实际结果**（实际发生了什么）
   - 示例："按钮点击后无反应，页面停留在登录界面"

### Step 2: 收集可选信息

根据需要询问：

6. **环境信息**（可选）
   - 操作系统、浏览器版本、运行时环境等
   - 示例："macOS 14.3, Chrome 122.0"
   
7. **错误日志**（可选）
   - 控制台错误、堆栈跟踪等
   - 如果内容较长，建议使用折叠方式展示

## 执行创建

信息确认后，执行脚本创建 Issue：

```bash
./scripts/create-bug.sh \
  "<问题标题>" \
  "<问题描述>" \
  "<复现步骤>" \
  "<期望结果>" \
  "<实际结果>" \
  "<环境信息>" \
  "<错误日志>"
```

## 输出格式

创建的 Issue 将包含以下结构化内容：

```markdown
## 问题描述
[用户输入的描述]

## 复现步骤
[步骤列表]

## 期望结果
[期望行为]

## 实际结果
[实际行为]

## 环境信息
[环境详情]

## 错误日志
<details>
<summary>展开查看</summary>
[错误日志内容]
</details>
```

Issue 会自动添加 `bug` 标签，标题前缀为 `[Bug]`。

## 前置要求

- GitHub CLI (`gh`) 已安装并登录
- 当前目录是 Git 仓库，且关联了 GitHub 远程仓库

## 错误处理

如果创建失败：
1. 检查 `gh auth status` 确认已登录
2. 检查当前目录是否为 Git 仓库：`git remote -v`
3. 确认有仓库写入权限
4. 检查脚本是否有执行权限：`chmod +x ./scripts/create-bug.sh`

## 示例对话

**用户**：记录一个登录页面的 bug

**助手**：好的，我来帮您创建 Bug Issue。请回答以下问题：

1. **问题标题**（一句话描述）：登录按钮点击无响应

2. **问题描述**：用户在登录页面输入正确的账号密码后，点击登录按钮没有任何反应

3. **复现步骤**：
   1. 访问 /login 页面
   2. 输入测试账号 test@example.com / 123456
   3. 点击"登录"按钮
   
4. **期望结果**：登录成功后跳转到首页 /dashboard

5. **实际结果**：按钮点击后无反应，页面停留在登录界面，控制台显示错误

6. **环境信息**（可选）：macOS 14.3, Chrome 122

7. **错误日志**（可选）：TypeError: Cannot read property 'token' of undefined

信息收集完成后，将执行脚本创建 Issue。
