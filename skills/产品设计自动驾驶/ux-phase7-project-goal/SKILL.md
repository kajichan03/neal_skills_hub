## 简单 vs 复杂功能点判断标准

### 简单功能点（流程演示 Demo 输出完整规则）

满足以下**全部**条件：

| 条件 | 说明 |
|------|------|
| 业务逻辑单一 | 没有复杂的分支判断 |
| 状态流转简单 | ≤2 种状态 |
| 异常处理简单 | 不涉及复杂的异常处理场景 |
| 数据规则明确 | 数据格式和校验规则明确 |
| 描述边界清晰 | 可以清晰描述「是什么」而不涉及「怎么实现」 |

**示例**：泳道新建表单的字段展示规则、按钮的显示/隐藏逻辑

---

### 复杂功能点（流程演示 Demo 输出摘要，项目章程 详细设计）

满足以下**任一**条件：

| 条件 | 说明 |
|------|------|
| 多步骤流程 | 涉及多步骤流程或状态机（≥3 种状态） |
| 复杂分支 | 有复杂的分支判断（if-else 嵌套） |
| 多系统交互 | 涉及多个系统的数据交互 |
| 异常流程 | 需要定义异常处理流程 |
| 权限逻辑 | 有复杂的权限判断逻辑 |
| 数据校验 | 需要明确定义复杂的数据格式和校验规则 |

**标记规则**：在功能点标题后添加 🔍，格式如 `【功能点】工单状态流转 🔍`

**示例**：流程日志的状态流转、重新部署的异步处理逻辑

### Step 4：拆解需求与功能点（必须关联功能模块）

使用 **AskUserQuestion tool** 与用户协作，确认需求归类：

> "基于 MVP Must-have 清单，我来帮你拆解为需求和功能点。
>
> **Must-have 清单**：
> - [列出所有 Must-have 项]
>
> **建议按以下业务场景归类**：
> 1. **[场景1]** - 包含：[项1, 项2]
> 2. **[场景2]** - 包含：[项3, 项4]
>
> **重要**：每个功能点必须归属于 `05-user-journey.md` 中定义的功能模块。
> 请确认或调整归类方式，并标注每个功能点所属的功能模块。"

### Step 5：逐个功能点定义规则（渐进式 + 功能模块关联）

对每个功能点，按以下流程定义规则：

**第一步：判断复杂度**

根据「简单 vs 复杂功能点判断标准」，确定该功能点属于哪种类型：
- **简单功能点**：在 流程演示 Demo 输出完整规则
- **复杂功能点**：在 流程演示 Demo 输出规则摘要，标记 🔍 留待 项目章程 详细设计

**第二步：定义规则内容**

对于**简单功能点**，需明确：
- **业务规则**：清晰描述规则项和规则内容
- **状态流转**：如涉及，列出所有状态及其转换条件
- **边界条件**：明确输入范围、校验规则、异常处理

对于**复杂功能点**，需明确：
- **规则摘要**：简述核心逻辑，标注为「待 项目章程 详细设计」
- **关键问题**：列出需要深入分析的点（如状态机、分支逻辑、数据交互等）
- **标记 🔍**：在功能点名称后添加标记

**第三步：关联功能模块**

每个功能点必须标注所属的功能模块（来自 05-user-journey.md），格式：
```
**功能模块**：[路径] — 来自 05-user-journey.md 的功能模块
```

### Step 6：整合并生成项目章程

将所有收集的信息整合为 `06-project-goal.md`：

1. **检查完整性**：确保所有 Must-have 都有对应的需求和功能点
2. **检查一致性**：确保功能模块关联正确，无遗漏
3. **检查复杂度分布**：统计简单/复杂功能点数量
4. **生成文档**：按项目章程结构模板输出

### Step 7：检查并确认

使用 **AskUserQuestion tool** 进行最终检查：

> "**Project Goal 初稿已完成！** 请检查以下内容：
>
> **角色定义**：[N] 个角色，是否完整？
>
> **需求覆盖**：是否覆盖了所有 Must-have？
>
> **功能点规则**：
> - 简单功能点（完整规则）：[N] 个
> - 复杂功能点（标记 🔍）：[N] 个
>
> **功能模块关联**：所有功能点是否都已标注所属功能模块？
>
> 是否有遗漏或需要调整？"

**迭代确认流程**：
1. 如有修改意见，更新文档
2. 再次检查，直到用户确认通过
3. 用户确认后，流程演示 Demo 完成

## 护栏（行为准则）

| 规则 | 说明 |
|------|------|
| **不包含技术实现细节** | 项目章程说「做什么」，不说「怎么做」 |
| **简单功能点必须有完整规则** | 不能遗漏关键业务逻辑 |
| **复杂功能点必须标记 🔍** | 明确告知哪些需要后续详细设计 |
| **功能模块必须关联** | 每个需求/功能点都必须标注所属功能模块（来自 `05-user-journey.md`） |
| **页面结构必须关联** | 每个功能点都必须标注涉及的页面结构（来自 05-user-journey.md Part 3） |
| **保持可追溯** | 每个需求/功能点都能追溯到 `05-user-journey.md` 的功能模块和页面结构 |
| **检查必须通过** | 必须经过用户确认才能进入下一阶段 |
