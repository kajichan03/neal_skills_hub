# 文档一致性检查工具

本工具用于在产品设计流程中保持 Context 文档的时效性，确保产品设计各阶段的输出能够及时同步到统一的上下文中。

## 核心职责

**唯一职责**：检测文档变更 → 提取关键信息 → 同步至 Context

## 工作流程

### Step 1：检测文档变更

识别最近更新或需要同步的文档：

1. 检查 `output/` 目录下的最新文件
2. 或根据用户指令明确指定需要同步的文档
3. 确认文档类型（Context、Project Goal、MVP Scope、PRD、Design 等）

### Step 2：提取关键信息

读取文档内容，根据文档类型提取对应的关键信息：

| 文档类型 | 需要提取的关键信息 |
|---------|-------------------|
| Context | 用户画像、场景描述、痛点清单、技术约束条件 |
| Project Goal | 目标描述、衡量指标、关键规则、优先级 |
| MVP Scope | 功能范围、优先级排序、核心功能清单 |
| PRD | 功能详情、业务流程、验收标准、边界条件 |
| Design | 界面方案、交互规则、视觉规范、组件列表 |
| User Journey | 用户旅程、触点分析、情感曲线、改进机会 |

> **注意**：仅提取核心要点，避免复制完整文档内容。

### Step 3：同步到 Context

将提取的关键信息更新到 Context 文档（如 `00-context.md` 或其他指定的 Context 文件）：

```markdown
## [文档类型] 同步信息

### 来源
- 文档：[文档名称]
- 更新时间：[YYYY-MM-DD]
- 文档路径：[相对路径]

### 关键信息
- [核心要点1]
- [核心要点2]
- [核心要点3]
...
```

**同步规则**：
- 如果目标 Context 文档不存在，创建基础结构
- 如果已存在对应同步区块，覆盖更新（幂等操作）
- 保持现有 Context 文档结构不变，仅追加同步区块

### Step 4：验证与确认

完成同步后，输出确认信息：

> "✅ 已将 [文档名称] 的关键信息同步到 Context。
>
> **同步内容摘要**：
> - [关键要点1]
> - [关键要点2]
> - [关键要点3]
>
> **Context 文档位置**：`[路径]`"

## 护栏规则

1. **仅同步关键信息**
   - 不复制完整文档内容
   - 只提取核心要点和关键结论

2. **自动触发机制**
   - 文档更新后自动执行
   - 可配置为后置钩子自动触发

3. **幂等操作**
   - 多次同步同一文档时覆盖而非追加
   - 确保 Context 内容始终为最新版本

4. **最小化干扰**
   - 不修改现有文档结构
   - 只在指定位置追加同步区块

5. **报告输出规范**
   - 如果生成任何 .md 格式的报告，**必须在开头提供目录（Table of Contents）**
   - 目录应包含报告的主要章节和页码/锚点链接

## 输出示例

### 成功同步示例

```
✅ 已将 ux-phase7-project-goal.md 的关键信息同步到 Context。

同步内容摘要：
- 目标：提升用户下单转化率 15%
- 衡量指标：下单转化率、客单价、复购率
- 关键规则：限时优惠、会员优先

Context 文档位置：output/00-context.md
```

## 适用场景

- 产品设计流程中任意阶段文档更新后
- 需要保持多文档间信息一致性时
- 作为自动化工作流的组成部分
