## 二、垂直领域知识库

### 2.1 优秀简历的核心标准

根据对 HR 筛选简历关注点的深度调研，优秀简历应满足以下六大核心标准：

| 评估维度 | 具体要求 | 重要性说明 |
|----------|----------|-------------|
| **完整性** | 包含基本信息、联系方式、教育背景、工作经历、技能证书等必要模块 | 缺失任意模块可能导致直接淘汰 |
| **符合度** | 内容与目标岗位要求高度匹配，关键词精准对应 | ATS 系统和 HR 人工筛选的核心依据 |
| **工作经验** | 描述清晰有条理，体现与岗位的相关性和能力质量 | HR 平均浏览仅 6 秒，是最抓眼球的内容 |
| **教育背景** | 学历、学校、专业与岗位要求匹配 | 硬性条件筛选的重要依据 |
| **技能匹配度** | 技能清单与岗位需求高度吻合 | 决定候选人是否通过初筛 |
| **软实力展现** | 语言能力、沟通能力、团队协作能力等体现 | 影响面试官的整体印象 |

### 2.2 HR 关注点的优先级排序

| 优先级 | 关注内容 | 停留时间 | 决策影响 |
|--------|----------|----------|----------|
| **第一优先级** | 工作经验 / 项目成就 | 6-15 秒 | 决定是否继续阅读 |
| **第二优先级** | 教育背景 / 技能匹配 | 3-5 秒 | 判断硬性条件 |
| **第三优先级** | 职业稳定性 / 发展轨迹 | 2-3 秒 | 评估候选人可靠性 |
| **第四优先级** | 自我评价 / 求职意向 | 1-2 秒 | 辅助判断匹配度 |

### 2.3 STAR 法则完整应用规范

STAR 法则是人力资源领域公认的简历描述方法，能够帮助求职者清晰、有条理地展示自己的工作经历和能力。以下是 STAR 法则在简历撰写中的详细应用：

| 要素 | 定义 | 简历应用技巧 | 优秀示例 |
|------|------|--------------|----------|
| **S - Situation 情境** | 项目或任务的背景挑战 | 用一句话描述所处的环境和面临的困难 | "在用户留存率持续下降 15% 的背景下" |
| **T - Task 任务** | 需要达成的具体目标 | 明确量化的目标值或预期成果 | "需要将 30 日留存率从 22% 提升至 35%" |
| **A - Action 行动** | 采取的具体措施 | 突出个人贡献，避免团队模糊表述 | "搭建用户分层运营体系，设计流失预警模型" |
| **R - Result 结果** | 量化的成果 | 用数据说话，与情境形成呼应 | "将 30 日留存率提升至 38%，超额完成目标 23%" |

**简历描述优化原则**：

- 每条成就描述控制在 50-80 字，2-3 句话为宜
- 必须使用强动词开头：主导、负责、推动、搭建、优化、实现、提升、构建
- 坚决删除弱动词：协助、参与、支持、帮忙、了解、接触
- 量化数据优先呈现：百分比、具体金额、影响人数、时间周期
- 自然嵌入 1-2 个目标岗位 JD 的关键词

### 2.4 ATS 系统关键词优化策略

ATS（Applicant Tracking System，招聘追踪系统）是现代企业初步筛选简历的核心工具。根据调研，约 80% 的大型企业使用 ATS 系统进行简历初筛。优化策略如下：

| 优化维度 | 具体方法 | 实施要点 |
|----------|----------|----------|
| **关键词提取** | 从目标岗位 JD 中识别高频词汇 | 识别出现频次 ≥2 次的硬技能、工具名称、行业术语 |
| **关键词匹配** | 使用 JD 原文术语，不使用近义词替代 | 如 JD 写 "Python"，简历中也写 "Python" 而非 "编程语言" |
| **关键词分布** | 在不同模块合理分布关键词 | 工作经历、技能清单、项目描述各出现 1-2 次 |
| **ATS 友好格式** | 采用标准排版，避免复杂元素 | 使用标准字体、避免表格和文本框、纯文本描述 |

## 三、Skill 结构设计

### 3.1 多步骤工作流架构

简历撰写与优化 Skill 采用五阶段渐进式工作流设计，遵循"渐进式披露"（Progressive Disclosure）的核心理念，避免一次性要求用户提供大量信息：

```
┌─────────────────────────────────────────────────────────────────┐
│                     简历撰写与优化工作流                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  阶段一：信息收集 ──→ 阶段二：岗位分析 ──→ 阶段三：内容优化     │
│        │                │                  │                   │
│        │                ▼                  ▼                   │
│        │           可选阶段           可选阶段                  │
│        │                │                  │                   │
│        ▼                ▼                  ▼                   │
│  基础信息表单        JD 分析           STAR 重写                │
│  工作经历表单       匹配度报告        量化转化                  │
│  项目经历表单       优化建议          弱点转化                  │
│  教育技能表单                                               │
│  目标岗位表单                                               │
│        │                                                     │
│        ▼                                                     │
│  阶段四：格式适配 ──→ 阶段五：输出生成                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 各阶段详细设计

#### 阶段一：信息收集（5 个子表单）

通过分步表单引导用户提供信息，每个表单控制在合理字段数量内：

| 步骤 | 表单名称 | 字段数量 | 输入类型 | 必填项 |
|------|----------|----------|----------|--------|
| 1.1 | 基本信息 | 10 个 | 文本 + 单选 | 是 |
| 1.2 | 工作经历 | 每段 6 个字段 | 文本 + 多行文本 + 单选 | 是 |
| 1.3 | 项目经历 | 每项 6 个字段 | 文本 + 多行文本 + 单选 | 否（建议 2-3 项） |
| 1.4 | 教育背景与技能 | 8 个 | 文本 + 多选 | 是 |
| 1.5 | 目标岗位 | 7 个 | 文本 + 单选 + 多选 | 是 |

#### 阶段二：岗位分析（可选阶段）

当用户选择「岗位定制」模式时启用此阶段：

| 任务 | 输出内容 | 说明 |
|------|----------|------|
| JD 关键词提取 | 分类关键词列表 | 硬技能 / 软技能 / 行业术语 / 职能动词 |
| 隐藏要求识别 | 隐性偏好和软性能力清单 | JD 中未明确写出但实际要求的要素 |
| 匹配度分析 | 可视化匹配度报告 | 用户经历与 JD 的契合程度评估 |
| 优化建议 | 具体改进方向 | 缺失关键词补充方向 |

#### 阶段三：内容优化

| 优化任务 | 应用方法 | 输出内容 |
|----------|----------|----------|
| 工作经历优化 | STAR 法则全面重写 | 每段经历生成 3 条成就描述 |
| 项目经历优化 | STAR 法则 + 个人贡献突出 | 完整的项目背景、职责、成果 |
| 成就量化转化 | 模糊描述 → 量化数据 | 全部可量化的成就陈述 |
| 弱点正向转化 | 职业空窗期、频繁跳槽等场景 | 提供多种正向表述方案 |
| 关键词嵌入 | JD 关键词自然融入内容 | 优化后保留关键词的内容版本 |

#### 阶段四：格式适配

| 检查维度 | 检查项目 | 优化建议 |
|----------|----------|----------|
| ATS 兼容性 | 关键词分布、格式纯净度 | 确保关键词出现频次合理，避免格式导致解析失败 |
| 排版规范 | 字体、字号、行距、边距 | 推荐：宋体 / 微软雅黑，10-11pt 正文，1.15-1.5 倍行距 |
| 模块结构 | 标题层级、留白、对齐 | 一级标题 14pt，二级标题 12pt，正文 10-11pt |
| 文件格式 | 输出格式选择 | 优先推荐 PDF 格式 |

#### 阶段五：输出生成

根据用户选择的优化类型，生成对应内容：

| 优化类型 | 适用场景 | 输出文件 |
|----------|----------|----------|
| **全新撰写** | 从零开始创建简历 | 完整结构化简历 + 求职信模板 |
| **现有优化** | 优化已有简历 | 优化对比报告 + 修改后简历 |
| **岗位定制** | 针对特定岗位定制 | JD 分析报告 + 定制简历 + 求职信 |
| **ATS 优化** | 重点提升通过率 | ATS 评估报告 + 格式优化建议 + 修改后简历 |

## 七、使用场景示例

### 场景一：应届生全新撰写

**用户输入信息**：

- 姓名：王小明
- 学历：本科，北京大学，计算机科学
- 工作年限：应届生
- 目标岗位：后端开发工程师
- 目标行业：互联网

**工作流程**：

1. 引导填写基本信息（姓名、学历、城市等）
2. 引导填写教育背景（GPA、相关课程、获奖情况）
3. 引导填写项目经历（毕设、课程项目、开源贡献等）
4. 引导填写技能清单（编程语言、框架、工具）
5. 选择优化类型（全新撰写）

**输出内容**：完整结构化简历 + 求职信模板

---

### 场景二：社招人员岗位定制

**用户输入信息**：

- 现有简历内容（粘贴全文）
- 目标岗位 JD（粘贴完整招聘信息）

**工作流程**：

1. 分析 JD，提取关键词列表
2. 对比用户经历与 JD 的匹配度
3. 生成匹配度分析报告
4. 针对 JD 要求优化简历内容
5. 生成定制化简历和求职信

**输出内容**：

- JD 关键词提取报告
- 匹配度可视化分析图表
- 定制化简历（Markdown + PDF）
- 定制化求职信

### 场景三：ATS 优化

**用户输入信息**：

- 现有简历（粘贴全文）

**输出内容**：

### 10.1 强动词清单（简历推荐使用）

| 类别 | 推荐动词 | 使用场景 |
|------|----------|----------|
| 管理类 | 主导、负责、统筹、管理、带领、协调、监管 | 团队管理、项目负责 |
| 规划类 | 规划、设计、搭建、构建、制定、建立、开发 | 系统建设、方案制定 |
| 优化类 | 优化、改进、提升、改善、增强、提高、推动 | 效率提升、业绩增长 |
| 成果类 | 实现、完成、达成、产出、交付、落地、执行 | 项目交付、目标达成 |
| 分析类 | 分析、调研、洞察、挖掘、评估、验证、追踪 | 数据分析、市场调研 |
| 协作类 | 对接、联动、协同、推进、整合、统筹 | 跨部门合作、资源整合 |

### 10.2 弱动词清单（简历应避免使用）

| 应删除的动词 | 原因分析 | 替代建议 |
|--------------|----------|----------|
| 协助 | 角色不明确，未体现个人贡献 | 改用「配合」「支持」并说明具体贡献 |
| 参与 | 贡献程度不明，无法体现价值 | 改用「主导」「负责」「推动」 |
| 支持 | 过于被动，未体现主动性 | 改用「响应」「满足」「保障」 |
| 帮忙 | 口语化，不专业 | 改用具体动词，如「协助」「协调」 |
| 了解 | 程度模糊，未体现掌握深度 | 改用「熟悉」「掌握」「精通」 |
| 接触 | 缺乏具体行动描述 | 改用具体工作内容描述 |
| 做过 | 表述笼统，缺乏专业性 | 改用具体动词和结果描述 |

### 10.3 量化指标参考词典

| 成果类型 | 可量化维度 | 示例表述 |
|----------|------------|----------|
| 效率提升 | 时间、成本、资源节省 | "处理效率提升 40%"、"节省 2 小时 / 天"、"降低人力成本 25%" |
| 业绩增长 | 销售额、用户数、转化率 | "销售额增长 30%"、"新增用户 10 万 +"、"转化率提升 15%" |
| 项目交付 | 按时完成率、质量 | "100% 按时交付"、"0 重大事故"、"缺陷率降低 60%" |
| 团队贡献 | 人员规模、管理幅度 | "带领 8 人团队"、"培训新员工 20 人 +"、"优化团队协作流程" |
| 影响范围 | 用户规模、业务范围 | "服务用户 100 万 +"、"覆盖业务线 5 条"、"日活 50 万" |
| 质量提升 | 准确率、满意度 | "准确率达 99.5%"、"客户满意度提升至 95%"、"投诉率下降 45%" |
