# skill: neal-core

把一段中文写得更短、更顺、更真。
重点不是“润色”。重点是搞清楚到底想说什么，然后说清楚。

## 红线（每条必须过，顺序即优先级）

1. 口语检验 — 最高法则。读出声来，你会这样跟一个聪明的朋友说话吗？不会→改到会。连词不是敌人——"但是""所以"是思维转弯的声音，只砍机械连词（"此外""值得注意的是"）
2. 零术语 — 聪明的 12 岁孩子能复述。专业词必须出现时，先用大白话把意思落地，再顺带提术语名
3. *短词优先* — 能用两个字说的不用四个字。「进行分析」→「看」。大词不让你显得聪明，只让人读得累
4. *一句一事* — 每句只推进一步。长句拆短
5. *具体* — 名词看得见，动词有力气。「有人觉得情况不太好」→「张三说项目要黄了」。形容词能砍就砍
6. *开头给理由* — 第一句话让人想读下一句。不铺垫、不背景、不「自古以来」
7. *不填充* — 删开场白、拐杖词、夸大象征。每句都在干活
8. *信任读者* — 跳过软化、辩解、手把手引导。说一遍够了
9. *诚实* — 想不清楚就说想不清楚。"大概 70%" 比"可能"诚实

## 工具箱（可以拿的工具，没有哪个是必须的）

- 类比 — 承重的，方法的关键组件都能映射上。沿着类比走一遍方法
- 餐巾纸速写 — 「以前这么想，现在应该这么想」的并排对比
- 好问题 — 把想要解决的困境变成一个让外行也好奇的问题
- 递进例子 — 从简单到复杂，一步步搭建理解
- 反问入链 — 遇到隐含假设，用问题打开
- 画面 — 闭眼能看到的场景。硬造的画面比没有更糟
- 骨架图 — 概念涉及空间关系时，嵌入 ASCII 图
- 故事 — 一个具体的人遇到一个具体的问题。读者跟着走
- 裂缝 — 模型/类比在哪里不够？那个点往往最值钱。不宣布它，让读者自己感到

## 执行

### 1. 探索想表达什么

一个人在想事情，碰巧被你看见。你和他是同行——走过类似的路，犯过类似的错，只是你先拐了这个弯。使用苏格拉底式的质疑，和他一起雕刻出那个具象的核心观点（一张图、一个场景、一个动作）。

#### 1. 找核

直接找出或者概括输入内容的核心观点——所有片段/概念都应该是围绕它生长的。

#### 2. 雕核

找到这个核心观点后，攻击这个核心观点，尝试不断提出问题使它陷入矛盾之中。：
* 反转："如果这是真的，那为什么……？"。
* 追问前提：“这个判断站在什么假设上？”
* 追问情绪：“为什么这件事让人不舒服/兴奋/困惑？”情绪指向未被说出的认知冲突。
* 翻转定义：“大家默认 A 是 A——但如果 A 其实是 B 呢？”。「品味」其实是「经验的积累」，「和谐」其实是「回避冲突」，「忙」其实是「不敢停下来」。

三种结果：
- 核心扛住了 → 带着更强的置信度往下走。
- 核心变形了，但也更坚固了 → 这是真正想要说的东西。
- 核心碎了 → 告诉用户：核心观点不成立，矛盾、问题在什么地方。

如果他在想两件事情，告诉他这是两件事情。
不确定就说不确定。"大概 70%" 比 "可能" 诚实。

### 2. 重塑表达

用雕琢后的核心观点去检查和重塑输入的内容。

如果有必要，重新组织输入的文章片段。片段应该以推理外显的方式组织 — 模拟"一个人想明白的过程"，而非呈现"想明白之后的结果"。用"既然A是B，那能不能C也是D？"带读者一起推。让读者觉得结论差一步就是自己想到的。

先亮自己的弯路，再给方向。这是所有说服力的来源——不是因为你对，而是因为你先错过。用自己踩坑的真实感换来对方放下防备，然后再说那句他需要听的话。

从工具箱里选最适合这个主题的方式，也可以不选——如果有更好的写法，用它。

如果有与中心无关的片段，去排除杂质。

禁止：
- 结构标签（`* 类比` / `* 裂缝 等）
- 指向写作过程的元评论（「打个比方」「接下来我们讨论」）
- 同一种句式结构最多出现一次。
- 借势：不用群体代言（"程序员都知道"）。不编造经历。不用元评论（"接下来我们讨论"）。

### 3. 过红线

逐条扫红线清单。额外检查：

- 破公式——否定式排比全文不超过两处，三段式改两项或四项
- 变节奏——长短句交替，段落结尾多样
- 杀金句——听起来像可引用的，重写
- 查跳跃——每步逻辑可追？前句说 A，后句跳到 B→补桥
- 查译感——动宾搭配中文天然吗？不自然→换动词或换句式

扫完列修改清单（哪句触发什么，改前→改后）。清单不写入文件。

### 4. 生成文件

在原目录原标题上加上“blowstone{日期时间}”。
