## 二、垂直领域知识库

### 2.1 优秀简历的核心标准

根据 HR 筛选简历的六大关注点，优秀简历应满足以下标准：

| 评估维度 | 具体要求 |
|----------|----------|
| 完整性 | 包含基本信息、联系方式、教育背景、工作经历、技能证书等必要模块 |
| 符合度 | 内容与目标岗位要求高度匹配，关键词精准对应 |
| 工作经验 | 描述清晰、有条理，体现与岗位的相关性和能力质量 |
| 教育背景 | 学历、学校、专业与岗位要求匹配 |
| 技能匹配度 | 技能清单与岗位需求高度吻合 |
| 其他因素 | 语言能力、沟通能力、团队协作能力等软实力体现 |

### 2.2 HR 关注的简历要素优先级

根据调研结果，HR 在筛选简历时的关注顺序如下：

```
第一优先级：工作经验 / 项目成就（HR 平均浏览简历仅 6 秒，这是最抓眼球的内容）
第二优先级：教育背景 / 技能匹配（硬性条件筛选）
第三优先级：职业稳定性 / 发展轨迹（判断候选人可靠性）
第四优先级：自我评价 / 求职意向（辅助判断匹配度）
```

### 2.3 STAR 法则应用规范

STAR 法则是 HR 最认可的简历描述方法，具体应用如下：

| 要素 | 定义 | 简历应用 | 示例 |
|------|------|----------|------|
| S - Situation 情境 | 项目或任务的背景挑战 | 一句话描述环境 | 在用户留存率持续下降的背景下 |
| T - Task 任务 | 需要达成的具体目标 | 明确的目标值 | 需要将 30 日留存率从 22% 提升至 35% |
| A - Action 行动 | 采取的具体措施 | 突出个人贡献 | 搭建用户分层运营体系，设计流失预警模型 |
| R - Result 结果 | 量化的成果 | 数据说话 | 将 30 日留存率提升至 38%，超额完成目标 23% |

**优化原则**：

- 每条描述控制在 50-80 字，2-3 句话
- 使用强动词开头：主导、负责、推动、搭建、优化、实现、提升
- 删除弱动词：协助、参与、支持、帮忙
- 量化数据优先：百分比、金额、人数、时间周期

### 2.4 ATS 系统关键词优化

ATS（Applicant Tracking System，招聘追踪系统）是大多数公司初步筛选简历的工具。优化策略如下：

| 优化维度 | 具体方法 |
|----------|----------|
| 关键词提取 | 从 JD 中识别出现频次 ≥2 次的硬技能、工具名称、行业术语 |
| 关键词匹配 | 使用 JD 原文术语，避免使用近义词替代 |
| 关键词分布 | 在不同模块（工作经历、技能清单、项目描述）合理分布 |
| ATS 友好格式 | 使用标准字体、避免复杂表格、纯文本描述 |

## 三、Skill 结构设计

### 3.1 多步骤工作流架构

简历撰写与优化 Skill 采用五阶段渐进式工作流：

```
┌─────────────────────────────────────────────────────────────────┐
│                        简历撰写与优化工作流                       │
├─────────────────────────────────────────────────────────────────┤
│  阶段一：信息收集 → 阶段二：岗位分析 → 阶段三：内容优化          │
│        → 阶段四：格式适配 → 阶段五：输出生成                      │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 各阶段详细设计

#### 阶段一：信息收集

通过交互式表单逐步引导用户提供信息，避免一次性要求大量输入：

| 步骤 | 收集内容 | 输入类型 | 必填项 |
|------|----------|----------|--------|
| 1.1 | 基本信息（姓名、学历、工作年限、联系方式等） | 文本 + 单选 | 是 |
| 1.2 | 工作经历（每段经历的公司、职位、职责、成就） | 多行文本 | 是 |
| 1.3 | 项目经历（2-3 个代表性项目） | 多行文本 | 否 |
| 1.4 | 教育背景与技能清单 | 文本 + 多选 | 是 |
| 1.5 | 目标岗位信息 | 文本 + 单选 + 多选 | 是 |
| 1.6 | 优化类型选择 | 单选 | 是 |

#### 阶段二：岗位分析（可选，仅定制场景启用）

当用户选择「岗位定制」模式时，分析目标岗位 JD：

- 提取核心关键词并分类（硬技能 / 软技能 / 行业术语 / 职能动词）
- 识别隐藏要求（隐性偏好、软性能力、价值观匹配）
- 对比用户经历与 JD 的匹配度
- 输出匹配度报告和优化建议

#### 阶段三：内容优化

| 优化任务 | 应用方法 | 输出内容 |
|----------|----------|----------|
| 工作经历优化 | STAR 法则重写 | 每段经历 3 条成就描述 |
| 项目经历优化 | STAR 法则 + 个人贡献突出 | 项目背景、职责、成果 |
| 成就量化转化 | 模糊描述 → 量化数据 | 可量化的成就陈述 |
| 弱点正向转化 | 职业空窗期、频繁跳槽等 | 正向表述方案 |
| 关键词嵌入 | JD 关键词自然融入 | 优化后的内容 |

#### 阶段四：格式适配

| 检查项 | 优化建议 |
|--------|----------|
| ATS 兼容性 | 关键词分布、格式纯净度 |
| 排版规范 | 字体、字号、行距、边距 |
| 模块结构 | 标题层级、留白、对齐 |
| 文件格式 | PDF 输出推荐 |

#### 阶段五：输出生成

根据用户选择的优化类型，输出对应内容：

- **全新撰写**：完整简历 + 求职信模板
- **现有优化**：优化对比报告 + 优化后简历
- **岗位定制**：JD 分析报告 + 定制简历 + 求职信
- **ATS 优化**：兼容性评估 + 优化建议 + 修改后简历

## 四、信息收集表单设计

### 4.1 基础信息表单

| 字段名称 | 字段类型 | 占位提示 | 说明 |
|----------|----------|----------|------|
| 姓名 | 文本 | 请输入您的真实姓名 | 必填 |
| 性别 | 单选 | 男 / 女 / 不便透露 | 选填 |
| 年龄 | 文本 | 如：28 岁或 1998 年生 | 选填 |
| 最高学历 | 单选 | 高中及以下 / 大专 / 本科 / 硕士 / MBA / 博士 | 必填 |
| 毕业院校 | 文本 | 请填写学校全称 | 必填 |
| 所学专业 | 文本 | 如：计算机科学与技术 | 必填 |
| 工作年限 | 单选 | 应届生 / 1 年以下 / 1-3 年 / 3-5 年 / 5-10 年 / 10 年以上 | 必填 |
| 所在城市 | 文本 | 如：北京市朝阳区 | 必填 |
| 联系电话 | 文本 | 请输入 11 位手机号 | 必填 |
| 电子邮箱 | 文本 | 请输入常用邮箱 | 必填 |

### 4.2 工作经历表单

每段工作经历需填写以下信息：

| 字段名称 | 字段类型 | 占位提示 | 说明 |
|----------|----------|----------|------|
| 公司名称 | 文本 | 请填写公司全称 | 必填 |
| 所属行业 | 文本 | 如：互联网 / 金融 / 咨询 | 必填 |
| 任职职位 | 文本 | 请填写职位名称 | 必填 |
| 在职时间 | 文本 | 格式：2020.03 - 2023.08 或 至今 | 必填 |
| 主要职责 | 多行文本 | 请简要描述您在该公司的主要工作内容 | 必填 |
| 关键成就 | 多行文本 | 请列举 2-3 项您最突出的工作成果，最好包含数据 | 建议填写 |

### 4.3 项目经历表单

选择 2-3 个最具代表性的项目：

| 字段名称 | 字段类型 | 占位提示 | 说明 |
|----------|----------|----------|------|
| 项目名称 | 文本 | 请填写项目名称 | 必填 |
| 项目时间 | 文本 | 格式：2021.06 - 2022.06 | 必填 |
| 项目背景 | 多行文本 | 请描述项目启动的原因、目标解决的问题 | 必填 |
| 您的角色 | 单选 | 项目负责人 / 核心成员 / 参与者 | 必填 |
| 具体职责 | 多行文本 | 请描述您在项目中的具体工作内容 | 必填 |
| 项目成果 | 多行文本 | 请量化项目成果，如：提升了 XX%、节省了 XX 成本 | 必填 |

### 4.4 目标岗位表单

| 字段名称 | 字段类型 | 占位提示 | 说明 |
|----------|----------|----------|------|
| 目标行业 | 多选 | 互联网 / 金融 / 咨询 / 制造 / 教育 / 医疗 / 快消 / 不限 | 必填 |
| 目标职能 | 文本 | 如：产品经理 / 数据分析 / 运营管理 | 必填 |
| 期望职位 | 文本 | 如：高级产品经理 / 数据分析专家 | 选填 |
| 期望薪资 | 文本 | 如：25K-35K / 面议 | 选填 |
| 目标公司类型 | 多选 | 外资企业 / 民营企业 / 国有企业 / 上市公司 / 创业公司 / 不限 | 选填 |
| 目标城市 | 文本 | 如：北京 / 上海 / 深圳 或 不限 | 选填 |
| 招聘 JD（可选） | 多行文本 | 请粘贴目标岗位的完整招聘信息，以便进行精准定制 | 选填 |

### 4.5 优化需求表单

| 优化类型 | 选项说明 | 输出内容 |
|----------|----------|----------|
| 全新撰写 | 从零开始创建简历 | 完整结构化简历 + 求职信模板 |
| 现有优化 | 对已有简历进行优化 | 优化对比 + 修改后简历 |
| 岗位定制 | 针对特定岗位定制 | JD 分析报告 + 定制简历 + 求职信 |
| ATS 优化 | 重点提升通过率 | ATS 评估报告 + 格式优化建议 |

1. ATS 兼容性评估报告
2. 关键词覆盖率分析
3. 格式优化建议

## 十、附录：参考资源

### 10.1 强动词清单

| 类别 | 推荐动词 |
|------|----------|
| 管理类 | 主导、负责、统筹、管理、带领、协调、监管 |
| 规划类 | 规划、设计、搭建、构建、制定、建立、开发 |
| 优化类 | 优化、改进、提升、改善、增强、提高、推动 |
| 成果类 | 实现、完成、达成、产出、交付、落地、执行 |
| 分析类 | 分析、调研、洞察、挖掘、评估、验证、追踪 |

### 10.2 弱动词清单（应避免）

| 应删除的动词 | 原因 |
|--------------|------|
| 协助 | 角色不明确，未体现个人贡献 |
| 参与 | 贡献程度不明，无法体现价值 |
| 支持 | 过于被动，未体现主动性 |
| 帮忙 | 口语化，不专业 |
| 了解 | 程度模糊，未体现掌握深度 |
| 接触 | 缺乏具体行动描述 |

### 10.3 量化指标参考

| 成果类型 | 可量化维度 | 示例表述 |
|----------|------------|----------|
| 效率提升 | 时间、成本、资源 | 提升 40%、节省 2 小时 / 天 |
| 业绩增长 | 销售额、用户数、转化率 | 增长 25%、新增 10 万用户 |
| 项目交付 | 按时完成率、质量 | 100% 按时交付、0 重大事故 |
| 团队贡献 | 人员规模、管理幅度 | 带领 8 人团队、管理 50 人 |
| 影响范围 | 覆盖用户、业务范围 | 服务 100 万 + 用户、日活 50 万 |
