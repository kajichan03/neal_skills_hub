# GitHub Feature Request 创建助手

通过交互式对话收集需求信息，创建格式规范的 GitHub Feature Request Issue。

## 触发条件

用户提到以下关键词时触发：
- "提交需求" / "创建 feature"
- "记录新功能" / "提个需求"
- "功能建议" / "新需求"
- "加个功能" / "需要实现"

## 信息收集流程

### Step 1: 收集核心信息

按顺序询问用户以下信息：

1. **需求标题**（一句话概括）
   - 示例："添加数据导出为 Excel 功能"
   
2. **现状**（当前是什么情况）
   - 示例："目前数据只能在系统中查看，无法导出到本地"
   
3. **问题/痛点**（遇到了什么问题）
   - 示例："用户需要离线分析数据，或者与其他系统共享数据时很不方便"
   
4. **预期**（希望达到什么效果）
   - 示例："支持将表格数据导出为 Excel (.xlsx) 和 CSV 格式"

### Step 2: 收集补充信息

根据需要询问：

5. **优先级**（可选）
   - 高 / 中 / 低
   - 示例："高 - 影响多个客户的使用"
   
6. **补充信息**（可选）
   - 建议的实现方案、参考材料、竞品参考等
   - 示例：
     ```
     建议方案：
     - 在数据表格右上角添加"导出"按钮
     - 支持选择导出格式（CSV/Excel）
     - 参考 Google Sheets 的导出体验
     ```

## 执行创建

信息确认后，执行脚本创建 Issue：

```bash
./scripts/create-feature.sh \
  "<需求标题>" \
  "<现状>" \
  "<问题>" \
  "<预期>" \
  "<优先级>" \
  "<补充信息>"
```

## 输出格式

创建的 Issue 将包含以下结构化内容：

```markdown
## 现状
[当前情况描述]

## 问题
[痛点描述]

## 预期
[期望的功能描述]

## 优先级
[优先级评估]

## 补充信息
[补充说明]
```

Issue 会自动添加 `enhancement` 标签，标题前缀为 `[Feature]`。

## 前置要求

- GitHub CLI (`gh`) 已安装并登录
- 当前目录是 Git 仓库，且关联了 GitHub 远程仓库

## 错误处理

如果创建失败：
1. 检查 `gh auth status` 确认已登录
2. 检查当前目录是否为 Git 仓库：`git remote -v`
3. 确认有仓库写入权限
4. 检查脚本是否有执行权限：`chmod +x ./scripts/create-feature.sh`

## 最佳实践

- **标题要简洁明确**：用一句话说清需求
- **现状要客观**：描述事实，不带情绪
- **痛点要具体**：说明什么场景下会遇到问题
- **预期要可验证**：便于后续验收

## 示例对话

**用户**：提个导出功能的需求

**助手**：好的，我来帮您创建 Feature Request。请回答以下问题：

1. **需求标题**：添加数据导出为 Excel 功能

2. **现状**：目前所有数据只能在网页上查看，无法下载到本地

3. **问题/痛点**：客户需要每月生成报表发给上级，现在只能手动复制粘贴，效率很低还容易出错

4. **预期**：在数据表格页面添加导出按钮，支持导出为 Excel 和 CSV 格式

5. **优先级**（可选）：高

6. **补充信息**（可选）：建议参考 Google Analytics 的导出交互体验

信息收集完成后，将执行脚本创建 Issue。
