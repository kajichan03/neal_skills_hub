# AI 执行能力金字塔

> 让Agent去创造工具，让工具去执行任务

## 金字塔结构

```
                    👤 人
                   /      \
                  /  设计  \
                 /  创造   \
                /__________\
               /            \
              /    🤖 Agent   \
             /   动态规划     \
            /   自主决策      \
           /__________________\
          /                    \
         /      🧩 Skills       \
        /     大模型泛化能力      \
       /      单一智能单元        \
      /__________________________\
     /                            \
    /         ⚙️ 脚本               \
   /       固定逻辑 确定性流程        \
  /       无判断 无意外情况           \
 /____________________________________\
```

### 循环方向

```
        ⬇️ 不断向下沉淀能力
        ⬇️ Agent 创造 Skills
        ⬇️ Skills 固化成脚本
        ⬇️ 脚本释放人的精力
        ↺ 回到顶层处理新问题
```

## 各层详解

### 🎯 最底层：脚本（Scripts）

**特征：**
- 逻辑完全固定
- 输入→处理→输出完全确定
- 不存在需要判断的地方
- 无意外情况

**示例：**
- 飞书自动化
- 数据定时同步
- 报表生成

**何时使用：** 能用脚本搞定的，绝不用Agent

---

### 🧩 中间层：Skills

**特征：**
- 需要大模型的泛化能力
- 单一、封装的智能单元
- 不需要自主决策和多步骤规划

**示例：**
- 资讯打分（重要性、相关性、时效性评估）
- 内容分类
- 情感分析

**核心价值：** 脚本搞不定，但不需要Agent那么重的场景

---

### 🤖 上层：Agent

**特征：**
- 动态规划能力
- 根据中间结果调整策略
- 无法提前规划所有步骤

**示例：**
- 竞品体验报告生成
- 复杂调研分析
- 多步骤决策任务

**使用原则：** 只处理那些还没法被固化的部分

---

### 👤 最顶层：人

**角色定位：**
- 定义目标
- 设计系统
- 处理真正的创造性工作
- 不断将经验沉淀为下层能力

## 核心原则

### ❌ 常见错误

```
┌─────────────────────┐
│        👤 人        │
│          ↓          │
│      🤖 Agent       │
│    ← 所有事情 →     │
│   慢、贵、不稳定    │
└─────────────────────┘
```

一上来就把所有事情都往Agent上堆，引入不必要的不确定性。

### ✅ 正确使用方式

```
        👤 人
         ↑
    释放精力，专注创造
         ↓
    🤖 Agent
         ↓
   动态规划，创造工具
         ↓
    🧩 Skills
         ↓
    泛化能力，封装单元
         ↓
    ⚙️ 脚本
         ↓
   固定逻辑，高效执行
         ↓
   循环回到 👤 人
```

**最佳实践：**
1. Agent 只处理还没法被固化的部分
2. 可复用的能力 → 沉淀为 Skill
3. 可固化的逻辑 → 下沉为脚本
4. 释放人的精力，专注于更高价值的工作

## 一句话总结

> **让Agent去创造工具，让工具去执行任务。**
> 
> 这个循环一旦跑起来，才是AI在公司里真正落地的样子。
